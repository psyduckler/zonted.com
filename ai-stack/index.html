The five harnesses

Each harness runs in parallel. Each owns the work it's best at.

  • openclaw + GLM 5.1 Social calendar, video builds
  • hermes + Codex 5.5 Coding, API infrastructure
  • Claude Code + Opus 4.7 Planning, Pinterest automation
  • AI Studio + Gemini 3.1 Pro Second-opinion planning
  • MiniMax 2.7 Low-think tasks (occasional)

Why five separate harnesses (instead of one orchestrator)

I could stitch all of this together with a single orchestration framework. I don't — each harness sits on its own token plan, so running them in parallel is meaningfully cheaper than fanning everything through a single API.

The other reason: the frontier is uneven. Different models lead in different domains. Opus is the best planner I've used. Codex is sharpest on API surface design. GLM 5.1 is shockingly cost-effective for high-volume content tasks. Gemini Pro pushes back hardest on plans I'm too confident about. Forcing all of those through one wrapper would homogenize the output and waste each model's edge.

The pattern

Plan with five models. Hand the chosen plan off to the one that's best at executing it.

A plan that survives review from Opus, GLM, Codex, and Gemini is a plan worth running. A plan that any one of them flags as wrong usually deserves another pass before execution.

Media generation

Wavespeed handles most video and audio. One API surface, frontier-model access across the board.

Native-API fallbacks for what Wavespeed doesn't proxy:

  • Gemini API — nano banana (image)
  • GPT API — image generation and Sora

Data & infrastructure

  • Cloudflare R2 — media storage (img.zonted.com)
  • Cloudflare Pages — site hosting
  • Cloudflare D1 — structured data when an agent needs a DB
  • Cron jobs — across each harness, the glue layer

No team. No orchestrator. Five harnesses on five token plans, sharing the same R2 bucket and D1 database.
